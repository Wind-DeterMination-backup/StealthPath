# Detail

## 这一层是什么
`tools/` 是仓库内部维护脚本层。

## 这一层具体干了什么
- 当前只有一个脚本：`generate_dox.py`
- 这个脚本负责扫描 git 跟踪文件，并为每个文件生成一个 `dox__*.md`

## 实现方式
- 先调用 `git ls-files` 获取文件列表。
- 对 Java 文件做轻量 API 提取：包名、类型名、public/protected 方法、public 字段。
- 对非 Java 文件只给出类型占位说明。
- 最后写出 `INDEX_dox.md` 作为映射索引。

## 与其他层级的关系
- 它不是游戏构建流程的一部分，而是维护者的文档辅助工具。
- 直接产物落在仓库根目录，而不是 `docs/`。

## 当前状态
- 脚本能自动产出文档骨架，但不会生成“目录层级说明”。
- 当前仓库里已经出现文档漂移，例如 `INDEX_dox.md` 对 `mod.json` 路径的认知仍停在旧布局，说明这个脚本或其产物没有在最近一次结构调整后完整刷新。
